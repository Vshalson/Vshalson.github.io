Automation with Ansible Playbooks

Automation

Learning ansible engine is not big task, big task is how to implement playbook for different environment based on requirement.

Suppose we have 100 servers and we need to install wget and vim to all servers

If we choose ansible, chef, puppet, salt or any diff tool we can complete task in 3 min.

  • Ansible will execute task on all servers parallely. So it will only 3 mins of time for any number of servers.
  • Playbook = Script to work with ansible
- hosts: all
	tasks:
	- yum:
			name: wget
			state: present
	- yum:
			name: vim
			state: present
  • Ansible will be helpful to perform:
    • Configuration management
    • application deployment
    • task automation
    • IT orchestration

Ansible architecture

Server which work with ansible called : Ansible Engine/Controller/Master

File containing information about remote servers : Inventory file

Passwords and other parameter containing file : Ansible.cfg

We connect to ansible nodes/ansible client

Playbook/Ad-Hoc commands are implemented using modules

Ansible uses SSH for operations

Ansible doesn't need anything installed in client side hence called agentless

Why ansible?

Uses existing openssh

Python/Yaml based syntax

Large number of ready to use modules for system management

Custom modules can be added if needed

  • configure ansible

Steps for Passwordless Authentication

  • Here create user ansadmin across all server to avoid confusion (real-time practice), in some organization they create extra user or sometime you use root and provide some passwords.
sudo su -
useradd ansadmin
passwd ansadmin
# Add user in Ansible Master and Ansible Node
  • Provide root privileges to all ansadmin users on all servers.
visudo OR /etc/sudoers.d
Add
ansadmin      ALL=(ALL)    NOPASSWD: ALL
  • Make sure that PasswordAuthentication Yes in all servers under /etc/ssh/sshd_config file and restart sshd server service ssh restart
  • Generate ssh-keys using ssh-keygen command from ansadmin
sudo su ansadmin
ssh-keygen          (enter.. enter..)
ssh-copy-id <public_ipv4>

# you can ssh using 
ssh <ipv4>
# repeat same for other ansible nodes in local server
ssh-copy-id <public_ipv4>
  • Update inventory with IPs. /etc/ansible/hosts

For ping if you get pong then it worked perfectly

If all your servers are hosted on cloud then you can use private IP.

Create backup of ansible config : ansible.cfg_bkp

Host key checking is parameter and for using ansible make sure to disable it.

Whenever you working with ansible in real world, create own directory for ansible instead of working in /etc/ansible 

# for exmaple
mkdir my_ansible_prod
# recursively copy all files
sudo cp -r /etc/ansible/* my_ansible_prod
sudo chown -R ansadmin *

Disable Host Key Checking 

# for temporary purpose
export ANSIBLE_HOST_KEY_CHECKING=False
# Permanent soltn
In ansible.cfg file write
host_key_checking = False

If you want to work with only one server give ipv4 name of that server but make sure that IP must be in your inventory file.

ansible <ipv4> -m ping
# to work with two server
ansible <ipv4_1>:<ipv4_2>:etc -m ping
